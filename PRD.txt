1. Executive Summary
Gaslight Guard is an AI-powered web platform that detects hidden verbal abuse patterns—sarcasm, passive-aggression, and gaslighting—in text conversations. Unlike standard sentiment analysis tools that misclassify polite-sounding toxic messages as positive, Gaslight Guard uses a fine-tuned RoBERTa model to perform multi-label classification across four categories: Sincere, Sarcastic, Passive-Aggressive, and Gaslighting.
The platform serves college students and young adults who experience subtle forms of verbal manipulation in relationships, friendships, academic environments, and online interactions. Beyond detection, it provides a Sanity Check validation system, suggested response templates, pattern tracking over time, and curated mental health resources.

2. Problem Statement
2.1 The Hidden Epidemic
Verbal abuse that hides behind politeness is one of the most psychologically damaging forms of manipulation. Victims often cannot articulate what feels wrong because the words seem normal on the surface. This phenomenon—commonly known as gaslighting—causes victims to doubt their own perception of reality.
2.2 Why Standard NLP Fails
Traditional sentiment analysis models evaluate text at surface level. A message like “Oh, you’re so smart for figuring that out all by yourself” would be classified as positive by most models due to words like “smart” and “figuring out.” In reality, this is a condescending, sarcastic remark designed to undermine confidence.
2.3 Target User Pain Points
	•	Students in toxic relationships who are told they are “overly sensitive”
	•	Individuals experiencing workplace/academic passive-aggression from authority figures
	•	People in manipulative friendships where backhanded compliments are the norm
	•	Anyone who needs an objective, AI-powered second opinion on whether communication is healthy

4.3 Model Specification
Base Model: RoBERTa-base (125M parameters)
Task: Multi-label text classification
Labels: Sincere, Sarcastic, Passive-Aggressive, Gaslighting
Training Data: SARC (Self-Annotated Reddit Corpus) + Cyberbullying Classification dataset
Inference: REST API hosted on Render with <500ms target latency

5.3 Dashboard (Authenticated Home)
The dashboard is the user’s command center—a personalized overview of their analysis history, patterns, and wellbeing.
5.3.1 Welcome Card
	•	Personalized greeting with name and date
	•	Quick-action buttons: New Analysis, View History, Resources
	•	Daily affirmation or mental health tip
5.3.2 Recent Analyses Widget
	•	Last 5 analyses with color-coded label badges
	•	Click-through to full analysis detail
5.3.3 Pattern Tracker
	•	Visual timeline/chart of toxicity patterns (7/30/90 days)
	•	Trend indicators: Improving, Stable, or Concerning

5.4 Analysis Page (Core Feature)
This is the heart of the product. The analysis page must feel powerful yet approachable—like talking to a trusted, intelligent friend.
5.4.1 Input Methods
	•	Text Input: Large textarea with character count and placeholder examples
	•	Conversation Mode: Multi-message input for entire threads
	•	Screenshot Upload: Drag-and-drop with OCR text extraction
5.4.2 Real-Time Analysis Display
	•	Animated loading state with shield icon
	•	Results card: Primary label, confidence score, secondary labels
	•	Sentence-by-sentence breakdown with inline highlighting in conversation mode
5.4.3 Sanity Check Module
The differentiating feature. After classification, the system generates human-readable validation:
	•	Gaslighting: “This follows a pattern of reality distortion. You are not being overly sensitive.”
	•	Passive-Aggressive: “While this sounds polite, the underlying tone carries hostility.”
	•	Sarcastic: “This uses sarcasm to deliver a negative message with plausible deniability.”
	•	Sincere: “This message appears genuine. No hidden toxicity detected.”
5.4.4 Suggested Responses
	•	Assertive: Sets a clear boundary
	•	De-escalation: Neutral redirect
	•	Disengage: Safe exit strategy
	•	Each template is copy-able with a single click

5.5 Conversation History Page
	•	Paginated list of all past analyses, sorted by date
	•	Filter by classification label
	•	Search by keyword across analyzed messages
	•	Bulk delete functionality
	•	Export as PDF report (for therapy sessions or documentation)

5.6 Pattern Insights Page
A dedicated analytics page surfacing long-term behavioral patterns.
	•	Toxicity Heatmap: Calendar view showing severity by day/week
	•	Relationship Tracker: Per-person toxicity trends (if messages are tagged by sender)
	•	Gaslighting Score: Aggregate 0–100 score based on frequency, severity, and consistency
	•	Contextual Consistency Report: 5-sentence sliding window analysis
	•	Exportable summary with charts for therapists/counselors

5.10 Bonus Features (Hackathon Differentiators)
These features go beyond core scope and serve as wow-factors for judges.
5.10.1 Anonymous Community Feed
An opt-in, anonymous feed where users share analyzed messages to normalize the experience. Users upvote with a “You’re Not Alone” reaction. Moderated via content filters. Builds community and reduces isolation.
5.10.2 Browser Extension Teaser
A landing section showing a future Chrome extension for in-context analysis on iMessage, WhatsApp Web, Instagram DMs, and Slack. Even as a mock-up, this demonstrates product vision and extensibility.
5.10.3 Therapist Shareable Report
One-click PDF report summarizing toxicity patterns, timeline, and severity scores. Bridges the gap between AI detection and professional support.
5.10.4 Tone Rewriter
After detecting a toxic message, rewrite the same sentiment in a healthy, direct way. Educational feature teaching users what healthy communication looks like.
5.10.5 Gamified Safety Score
Personal Safety Score (0–100) that improves as users engage with resources, log moods, and build healthier patterns. Achievement badges: Boundary Builder, Pattern Spotter, Self-Advocate. Gamification increases retention.

7. NLP Evaluation Plan
7.1 Sarcastic Sentiment Flip Test
Stress test using 100 curated positive-worded insults. Measures whether the model correctly identifies these as toxic rather than sincere. Target: >85% correct classification.
7.2 Contextual Consistency Audit
Evaluates gaslighting pattern detection across 5-sentence sliding windows. Tests whether the model identifies escalating manipulation across multiple messages. Target: >80% pattern detection accuracy.

8. Deployment & Infrastructure
8.1 Railway Configuration
	•	Framework: Vite (React) with npm run build
	•	Env vars: VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, VITE_API_URL

	•	Docker container with Python FastAPI serving RoBERTa
	•	GPU recommended; CPU fallback for hackathon
	•	CORS configured for Railway frontend origin
	•	Rate limiting: 60 requests/minute per user
8.3 Supabase Configuration
	•	Row Level Security (RLS) on all tables
	•	Auth: Email/Password + Google OAuth
	•	Storage bucket for screenshots (5MB limit)
	•	Edge Functions for OCR and report generation
